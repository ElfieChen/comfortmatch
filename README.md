# comfortmatch
a simple tool that turns your comfort answers into clear clothing recommendations with short “why” explanations.

**Short Status Note:**
The current MVP prototype includes only front-end functionality, built using HTML/CSS/JavaScript and running on a local JSON product catalog. Testers can complete 7 comfort questionnaires, view dynamically generated personal profile summaries, and receive personalized recommendation streams from a curated product catalog (currently containing 20 items). The recommendation stream is generated through filtering and sorting. Each recommendation card displays a product image, comfort level label, brief “reason for recommendation” description, and brand-neutral “online search/physical store search” links for next steps. The primary testing focus for the current system is: whether the questionnaire design is clear and easy to understand, and whether the recommendation results and explanations are accurate and credible.

**Tester Start Guide:**
1. Start at the questionnaire, answer the 7 questions, and submit to generate your profile and recommendations.
2. On the recommendations page, review the cards (images/tags/“why this fits”) and click “Search Online” or “Search Shopping” on one item to see the next-step flow.
3. Go back and change one or two answers and resubmit, then tell us whether the recommendations change in a way that matches your expectations.
